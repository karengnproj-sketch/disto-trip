2 Database Entity-Relationship Diagram

PostgreSQL schema on Supabase. 8 tables with Row Level Security (RLS) policies. Crow's foot notation for cardinality.

Row Level Security (RLS)

All tables have RLS enabled. Profiles are readable by the owner only. Saved places and reviews enforce user_id = auth.uid() for INSERT/UPDATE/DELETE. Hotels, attractions, cities, and emergency facilities are publicly readable.

Supabase Auth Trigger

A database trigger on auth.users automatically creates a corresponding row in the profiles table on user signup, using handle_new_user() function. The profiles.id is a foreign key referencing auth.users.id.

Polymorphic References

saved_places and reviews use a polymorphic pattern with place_type (enum: 'hotel' | 'attraction') and place_id (UUID) to reference either hotels or attractions without separate join tables.

JSONB Columns

attractions.opening_hours stores per-day schedules as JSONB. crowd_data stores hourly crowd estimates by day of week. Both support flexible querying via Supabase's .contains() and arrow operators.

Internationalization

All user-facing content columns have _ar suffixes for Arabic translations. The app reads the user's preferred_language from their profile and selects the appropriate column at query time.

Storage & CDN

Image URLs reference Supabase Storage buckets. Hotels use text[] arrays for multiple gallery images. Cities and attractions use single image_url fields. All served via Supabase CDN with automatic transforms.
